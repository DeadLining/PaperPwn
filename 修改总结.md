## 9. 备注

- Outline 的 AI 结果现在依赖缓存，PDF 自带 Outline 优先显示。
- MindMap 和 Outline 的 AI 生成都尽量做成“有缓存就直接显示，有需要时再重新生成”。
- 若遇到老缓存或旧数据结构，当前代码里已经加了部分兼容处理。

- `src/components/PdfViewer.tsx`
  - 继续修复原生 PDF.js Viewer：缩放重建页面时记录当前视口锚点页和页内比例，重建后恢复到同一阅读位置，避免放大/缩小后自动跳回第一页。
  - 修复滚动触发渲染时频繁递增 render sequence 导致旧任务不断失效的问题，降低快速滚动时的空白和卡顿概率。
  - 修复 Alt/Ctrl/Meta + 滚轮缩放过猛的问题：改用 ref 读取最新视觉缩放值，并进一步限制单次滚轮 delta，避免从 221% 直接跳到 25%。
  - 修复 fit-width 下视觉预缩放比例使用固定 zoom 计算的问题，改为基于实际已提交渲染 scale 计算临时 transform。
  - 点击 PDF 页面空白处时主动清理旧文本选区，降低误点空白导致选区残留/乱选的问题。
  - 打开 PDF 后不再强制把全局当前页重置为第 1 页，尽量保留 ReaderView 恢复的 lastPage。
  - 修复上一次改动遗漏的问题：加载 PDF 的 `useEffect` 不再依赖 `loadPdf` 函数身份，避免 `zoom` / `scaleMode` 改变时触发整份 PDF 重新加载，从根因上解决缩放后仍跳回第一页的问题。
  - 滚轮缩放改为以鼠标所在位置为锚点：记录鼠标下方对应的 PDF 页码、页内比例和视口 Y 坐标，重渲染后恢复到同一内容点，避免继续以页面上方为缩放中心。
  - 将 Legacy PDF.js Viewer 的滚轮缩放锚点从鼠标位置调整为视口中心，避免临时 CSS 缩放阶段页面先跟随鼠标偏移、重渲染后又跳回锚点造成的“先变后跳”观感。
  - 进一步修复缩放“先变位置、过会儿又跳回”的问题：移除缩放过程中的临时 CSS 视觉预缩放，滚轮只更新目标 zoom，由 debounced PDF.js 重渲染一次性切换到新尺寸，优先保证阅读位置稳定。
  - 新增 ReadPaper 风格平滑缩放层：滚轮缩放期间使用 `visualZoom` 做即时 CSS 缩放，并用外层 `zoomShell` 同步扩展滚动尺寸，避免 transform 后 scrollHeight 不变导致的回跳。
  - 滚轮缩放重新启用鼠标位置锚点：按鼠标下方的页码、页内比例和视口 Y 坐标补偿 `scrollTop`，让鼠标指向的内容尽量保持不动。
  - 将可见页判断和当前页计算改为按 `visualZoom / committedScale` 折算，保证视觉缩放期间滚动渲染和页码同步仍按正确内容位置计算。
  - 用户停止滚轮后再 debounce 提交真实 `zoom`，由 PDF.js 高分辨率重渲染替换临时视觉缩放。
  - 平滑缩放锚点从仅 Y 轴扩展为 X/Y 二维锚点：记录鼠标下方的 `xRatio`、`yRatio`、`viewportX`、`viewportY`，缩放时同时补偿 `scrollLeft` 和 `scrollTop`。
  - 放大到页面宽度超过容器时，横向滚动也会跟随鼠标位置补偿，鼠标指向的二维内容点更稳定。
  - 修复连续 wheel 过快时缩放状态交错的问题：新增 `zoomGestureRef` 将一串 wheel 识别为同一次缩放手势，手势期间固定初始 X/Y 锚点，只累积最终 `targetZoom`。
  - 新增 `zoomFrameRef` 用 `requestAnimationFrame` 合并高频 wheel，保证每帧最多应用一次视觉缩放和滚动补偿，避免多个 restore 任务排队造成位置错乱。
  - 缩放手势期间暂停滚动触发的可见页渲染，避免旧 scale 的懒渲染和视觉缩放互相抢任务。
  - wheel 停止 360ms 后再提交真实 `zoom` 并触发 PDF.js 高清重渲染；卸载时会清理缩放 RAF、timer 和手势状态。
